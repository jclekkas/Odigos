#122 — Bounded Self-Healing Error Recovery

Make concrete code changes in the existing Odigos codebase. This is an implementation task, not a planning task.

Objective
Introduce bounded, loop-safe self-healing across the app:
- Retry only transient failures
- Prevent retry storms and duplicate side effects
- Add circuit breaker protection for AI
- Replay DLQ safely
- Improve UX recovery

Do NOT redesign unrelated systems.

---

## 1) Shared reliability helpers

Create:
- server/lib/reliability.ts
- client/src/lib/reliability.ts (or shared if clean)

Export:
- isRetriableError(error | status)
- getRetryDelay(attempt)

Rules:
- Retriable: network failures, timeouts, 408, 429, 500, 502, 503, 504
- Non-retriable: 400, 401, 403, 404, validation errors
- Retry delay:
  - exponential backoff capped (~8s)
  - add jitter (randomized)

Replace any scattered retry logic with this.

## 3) AI retry wrapper (analyze route)

File:
- server/routes/analyze.ts

Wrap ONLY the OpenAI call:
- openai.chat.completions.create(...)

Behavior:
- max 2 retries (3 attempts total)
- retry ONLY transient errors
- per-attempt timeout
- exponential backoff with jitter
- respect total request time budget

DO NOT retry:
- validation
- prompt building
- persistence
- parsing errors

Log:

- error type

Return clean 502 if exhausted.

## 4) Circuit breaker for AI

Create:
- server/lib/circuitBreaker.ts

Features:
- states: CLOSED, OPEN, HALF_OPEN
- rolling window tracking

Open when:
- failure threshold exceeded (timeouts, 429, 5xx, network)

Do NOT count:
- validation errors
- parsing errors
- bad input

Behavior:
- OPEN → fail fast (no AI call)
- cooldown → HALF_OPEN
- allow limited probe requests
- success → CLOSE
- failure → OPEN again

Wrap AI call AFTER retry logic.

Log:
- open / half-open / close transitions

## 6) DLQ replay worker

Create:
- server/warehouse/dlqReplay.ts

Behavior:
- runs:
  - on startup
  - on interval (configurable)

Process:
- fetch rows:
  - status = pending
  - next_attempt_at <= now()
- lease row:
  - status = leased
  - set lease_expires_at
- attempt write
- on success:
  - mark resolved
- on failure:
  - increment attempt_count
  - set next_attempt_at (backoff)
  - update last_error fields
- if attempt_count >= max_attempts:
  - mark dead

Constraints:
- paginated fetch (no full table load)
- no overlapping runs (lock/guard)
- replay is non-fatal
