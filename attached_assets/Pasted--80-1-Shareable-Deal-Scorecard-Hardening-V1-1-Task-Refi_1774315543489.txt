#80.1 - Shareable Deal Scorecard Hardening (V1.1)

Task:
Refine the existing Shareable Deal Scorecard implementation to improve clarity, reliability, and UX without changing the overall feature scope.

This is a polish + correctness pass, not a redesign.

---

Objective:
Improve:
- issue readability (less mechanical, more human)
- data safety guarantees
- PNG rendering reliability (especially Safari)
- clipboard UX feedback

Do NOT change:
- feature scope
- placement
- free-tier availability
- backend (still frontend-only)

---

Scope (STRICT):
Modify only:
- client/src/components/ShareCard.tsx
- client/src/pages/home.tsx

Do NOT:
- add dependencies
- change routing
- touch backend
- modify pricing/paywall logic
- redesign layout

---

Tasks:

1) Improve Issue Label Quality (CRITICAL)

Current source:
missingInfo[].field

Problem:
These can feel mechanical or unclear (e.g., "apr_missing", "doc_fee_high")

Solution:
Introduce a mapping layer:

Create helper:
formatIssueLabel(field: string): string

Rules:
- convert snake_case / keys into readable phrases
- remove underscores
- capitalize properly
- map known fields to strong user-facing phrasing

Examples:

"apr_missing" → "APR is missing — total cost is unclear"
"doc_fee_high" → "Documentation fee appears unusually high"
"otd_missing" → "Full out-the-door price not provided"
"addons_unclear" → "Add-ons may not be optional"

Fallback:
- replace underscores with spaces
- capitalize first letter

IMPORTANT:
- keep output short (1 line)
- no numbers
- no dollar amounts

Use this function for BOTH:
- copied summary
- ShareCard display

---

2) Strengthen Data Sanitization

Enhance sanitizeIssueLabel():

Ensure removal of:
- dollar amounts ($123, 123.45)
- long numbers (4+ digits)
- percentages
- any raw numeric patterns

Regex requirements:
- strip currency patterns
- strip long numeric sequences
- trim whitespace

Final output must:
- contain NO dollar values
- contain NO exact numbers tied to the deal

---

3) Improve Clipboard UX

In home.tsx:

After "Copy summary":
- show lightweight success feedback:
  - temporary text change (e.g., "Copied ✓")
  OR
  - small inline message

Requirements:
- no toast library
- no dependency
- revert back after ~1.5–2 seconds

Also:
- handle clipboard failure gracefully (fallback message)

---

4) Harden PNG Rendering (IMPORTANT)

In ShareCard + html-to-image usage:

Ensure:
- explicit width/height: 1200x630
- solid background color (no transparent default)
- fonts are loaded before capture

Before capture:
- ensure component is mounted
- avoid display:none (use off-screen positioning instead)

Add small delay if necessary before capture (e.g., requestAnimationFrame or short timeout)

Ensure:
- no clipped text
- consistent padding
- badge colors render correctly

Test scenarios:
- long issue text
- 1 issue
- 3 issues
- zero issues

---

5) Zero-Issue State (EDGE CASE)

If no issues exist:

Display in BOTH:
- summary text
- scorecard

Use:

Issues flagged:
• No major issues surfaced in this summary

Ensure layout remains balanced (no empty space collapse)

---

6) Text Polish (IMPORTANT)

Update copied summary format to improve readability:

Odigos Deal Scorecard
Verdict: [GO / NO-GO / NEED-MORE-INFO]
[verdictLabel]

Issues flagged:
• [issue 1]
• [issue 2]
• [issue 3]

Analyzed by Odigos — odigos.replit.app

Add blank line before "Issues flagged"

Ensure:
- no trailing bullets
- no awkward spacing
- reads cleanly in iMessage / Slack

---

7) Visual Stability Checks

Ensure ShareCard:
- handles long text wrapping cleanly
- does NOT overflow container
- maintains consistent spacing across all states
- badge colors:
  - green = GO
  - yellow = NEED-MORE-INFO
  - red = NO-GO
