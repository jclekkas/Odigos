Implement product analytics using PostHog for Odigos.

This is a lightweight instrumentation task.

DO NOT change:

* backend logic
* pricing
* analysis logic
* routing behavior
* API payloads
* UI/UX

---

OBJECTIVE

Track the core funnel so we can measure user behavior and drop-off:

1. landing → analyze click
2. analyze page view
3. input started
4. analysis submitted
5. analysis completed
6. paid conversion (only if a clear client-side success event exists)

---

IMPLEMENTATION

1. Install and initialize PostHog (client-side only)

* Use env vars for:

  * POSTHOG_KEY
  * POSTHOG_HOST (default to https://app.posthog.com)

* Initialize once in the app root

* If env vars are missing, fail gracefully (no errors)

---

2. Create a small analytics utility

Example:
capture(eventName, properties)

* Safe no-op if PostHog is not initialized
* Do not scatter raw PostHog calls across code

---

3. Track the following events

Event: landing_cta_clicked
Trigger: clicking "Check a Dealer Quote"
Props:

* location: "hero"
* cta_text

---

Event: analyze_page_viewed
Trigger: /analyze page load
Props:

* route: "/analyze"

---

Event: analysis_input_started
Trigger: first user interaction with textarea (typing OR paste)
IMPORTANT:

* fire only once per page visit
  Props:
* input_method: "typing" or "paste"

---

Event: analysis_submitted
Trigger: submit action
Props:

* input_mode: "paste" or "upload" (if available)
* input_length_bucket: ["1-99","100-299","300-599","600+"]

DO NOT send actual text.

---

Event: analysis_completed
Trigger: successful result render (not loading, not error)
Props:

* result_available: true

---

Event: paid_conversion
ONLY IF a clear client-side success event exists

Props:

* product: "full_review"

If unclear, do not guess — note where it should go.

---

PRIVACY RULES

DO NOT track:

* pasted dealer messages
* uploaded files
* names, emails, phone numbers
* any user content

Only track behavior.

---

DONE LOOKS LIKE

* events fire correctly
* no console errors
* app works normally without PostHog keys
* minimal code footprint

---

RETURN

* files changed
* unified diff
* where each event is triggered
* env vars required
* whether paid_conversion was implemented or deferred
