Update the existing Odigos prototype (do NOT rebuild from scratch). The app currently analyzes pasted dealer text and returns a JSON-driven result with a GREEN/YELLOW/RED score, GO/NO-GO, missing info questions, and a suggested reply.

Goal of this update:
1) Remove contradictions like “GREEN” + “missing key terms” by adding an explicit confidence + verdict nuance.
2) Tighten language so Odigos never sounds overconfident when information is missing.
3) Make scoring more opinionated and consistent with clear heuristics.

Required changes:

A) Add two new output fields to the JSON schema + UI
- confidenceLevel: "HIGH" | "MEDIUM" | "LOW"
- verdictLabel: a short human label shown under the big score, such as:
  - "PROCEED — CONFIRM DETAILS"
  - "PAUSE — GET OTD BREAKDOWN"
  - "NO-GO — TOO MANY RED FLAGS"
Render verdictLabel directly under the big GREEN/YELLOW/RED score.

Update the TypeScript interface to include:
{
  dealScore: "GREEN" | "YELLOW" | "RED",
  confidenceLevel: "HIGH" | "MEDIUM" | "LOW",
  verdictLabel: string,
  goNoGo: "GO" | "NO-GO" | "NEED-MORE-INFO",
  ...
}

B) Update scoring heuristics (implement deterministically in code BEFORE/AFTER LLM as needed)
Introduce a lightweight rule engine that adjusts dealScore/goNoGo/confidenceLevel based on what’s present:

Key definitions:
- “OTD present” means estimatedOutTheDoor is a number.
- “Payment-only quote” means monthlyPayment present but salePrice AND estimatedOutTheDoor are missing.
- “Vague fees” means fees list contains items like "dealer fee", "doc fee", "market adjustment", "protection package", "nitrogen", "etch", "paint", "ceramic", "appearance package" without clear amounts OR with unusually high amounts.

Rules (apply in this order):
1) If payment-only quote (no salePrice and no OTD):
   - dealScore = "YELLOW"
   - goNoGo = "NEED-MORE-INFO"
   - confidenceLevel = "MEDIUM"
   - verdictLabel = "PAUSE — ASK FOR ITEMIZED OTD"
2) If OTD is missing entirely:
   - dealScore = "YELLOW"
   - goNoGo = "NEED-MORE-INFO"
   - confidenceLevel = "MEDIUM"
   - verdictLabel = "PAUSE — GET OTD BREAKDOWN"
3) If OTD present AND APR/term present BUT MSRP or salePrice missing:
   - dealScore = "GREEN"
   - goNoGo = "GO"
   - confidenceLevel = "MEDIUM"
   - verdictLabel = "PROCEED — CONFIRM DETAILS"
4) If OTD present AND APR/term present AND no red flags detected:
   - dealScore = "GREEN"
   - goNoGo = "GO"
   - confidenceLevel = "HIGH"
   - verdictLabel = "GO — TERMS LOOK CLEAN"
5) If market adjustment or clear add-ons are detected (even if OTD present):
   - dealScore = "YELLOW" (or "RED" if multiple)
   - goNoGo = "NEED-MORE-INFO" (or "NO-GO" if severe)
   - confidenceLevel = "LOW" or "MEDIUM"
   - verdictLabel = "PAUSE — REMOVE ADD-ONS / MARKUP"
6) RED conditions (set to RED + NO-GO + LOW confidence):
   - Any explicit “market adjustment”/“markup” that is non-trivial
   - Multiple high-cost add-ons
   - Dealer refuses to provide OTD in text (if detected)
   - Contradictory numbers (e.g., two different OTD prices)
   - verdictLabel = "NO-GO — TOO MANY RED FLAGS"

Note: keep this rule engine simple and transparent. The LLM can still provide reasoning, but your app must not output GREEN with HIGH confidence when important fields are missing.

C) Update the LLM instructions to reduce overconfident phrasing
In the LLM prompt, add strict language rules:
- Never say “all key terms are clear” if any of MSRP, sale price, or OTD is missing.
- If information is missing, explicitly say: “Major terms look good, but confirm X, Y, Z.”
- Praise positive signals explicitly (trust-building):
  - If OTD provided: “Providing an out-the-door price upfront is a positive transparency signal.”
  - If low APR: “Low APR meaningfully reduces total cost.”

D) UI copy improvements (small but important)
- Under “What This Deal Likely Means,” ensure the first paragraph follows this template:
  1) What we know (facts only)
  2) Why it’s good/bad (based on rules)
  3) What’s missing (if anything) + next questions
- Keep it short: 4–6 sentences max.

E) Update “Missing Information” section behavior
- Only show “Missing Information” if there are actually missing core fields OR confidenceLevel != "HIGH".
- If verdictLabel is “PROCEED — CONFIRM DETAILS,” limit missing questions to max 3.

F) Maintain safety and trust
- Never invent numbers.
- No “market average” claims without data.
- Keep tone calm and practical.

Deliverables:

  1) Good deal with OTD + APR/term but missing MSRP -> GREEN, MEDIUM confidence, “PROCEED — CONFIRM DETAILS”
  2) Payment-only quote -> YELLOW, MEDIUM confidence, “PAUSE — ASK FOR ITEMIZED OTD”
  3) Market adjustment/add-on heavy -> RED or YELLOW depending on severity, LOW confidence
