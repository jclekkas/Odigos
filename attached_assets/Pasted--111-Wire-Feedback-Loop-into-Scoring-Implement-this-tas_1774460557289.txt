1) EXTEND WAREHOUSE DEALER AGGREGATE WITH FEEDBACK STATS
File: server/warehouse/setupViews.ts

Extend the dealer-level aggregate / materialized view by joining deal_feedback to dealer_submissions via listing_id.

Add these fields:
- positive_feedback_count
- total_feedback_count
- feedback_agreement_pct

Rules:
- Only count explicit 'up' / 'down' feedback
- Never count null / missing feedback rows
- Use NULL-safe division
- Preserve all existing columns and existing dealer aggregate behavior
- Do not break view creation / refresh / rebuild behavior
- Keep metrics numeric in the warehouse layer; do not format as percentages there

Use this exact SQL pattern, adjusting only for schema specifics:

SUM(CASE WHEN df.feedback = 'up' THEN 1 ELSE 0 END) AS positive_feedback_count,
COUNT(CASE WHEN df.feedback IN ('up','down') THEN 1 END) AS total_feedback_count,
CASE
  WHEN COUNT(CASE WHEN df.feedback IN ('up','down') THEN 1 END) > 0
  THEN SUM(CASE WHEN df.feedback = 'up' THEN 1 ELSE 0 END)::float
       / COUNT(CASE WHEN df.feedback IN ('up','down') THEN 1 END)
  ELSE NULL
END AS feedback_agreement_pct
