Implement Task #18 — Final UX Polish + Structured Header for Odigos.

This is a small follow-up UX refinement pass after the previous polish release.

The goal is to resolve the final UI nitpicks identified in the production UX audit and improve the structural presentation of the Odigos logo.

This is NOT a redesign and NOT a feature project.

PRIMARY GOAL
Finish the last usability and credibility improvements:

1. Slight crowding of example buttons on mobile
2. Overly wide text blocks in analyzer results on large monitors
3. Missing breadcrumbs on article pages
4. Article prose width still too wide on desktop
5. Logo presentation feels visually “floating” — introduce a structured header container that anchors the interface

NON-NEGOTIABLE CONSTRAINTS

- Do NOT change OpenAI prompts
- Do NOT change analyzer logic
- Do NOT change scoring or rule engine
- Do NOT change Stripe flow or pricing
- Do NOT add features, accounts, or dashboards
- Do NOT change frameworks (remain Vite + Express)
- Do NOT add dependencies
- Do NOT materially rewrite page copy
- Do NOT modify server routes except where explicitly noted
- Preserve existing SEO metadata
- Preserve current dark theme and visual system

All work should be purely layout, typography, and component structure.

SUCCESS CRITERIA

• Mobile example buttons no longer feel cramped  
• Analyzer explanation text is readable on large screens (~700–800px width)  
• Article pages include simple breadcrumbs  
• Article prose is constrained to readable width (~65ch)  
• The analyzer page and article pages have a proper full-width header container  
• The logo no longer appears as a floating element on the page  
• The interface feels structurally grounded and calm
