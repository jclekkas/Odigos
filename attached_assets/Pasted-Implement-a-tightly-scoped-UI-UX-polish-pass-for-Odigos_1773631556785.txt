PRIMARY GOAL
Improve trust, readability, hierarchy, spacing, and visual polish so Odigos feels calm, credible, and professional — closer to a serious consumer-protection tool than an “AI playground.”

DESIGN DIRECTION
Odigos should feel:
- calm
- serious
- trustworthy
- document-centric
- polished
- stable

It should NOT feel:
- flashy
- crypto-like
- neon
- “AI toy”
- overly aggressive

GLOBAL DESIGN RULES
Apply these principles consistently across the touched pages/components:
- Remove or drastically reduce aggressive blue glow / halo effects
- Prefer subtle borders, clean surfaces, restrained contrast, and structured spacing
- Improve typography hierarchy and breathing room
- Use clearer button hierarchy:
  - primary CTA = filled / highest emphasis
  - secondary actions = outline or lower emphasis
  - low-priority links = text only
- Keep the UI calm and objective, especially around negative analysis outcomes
- Constrain long-form reading widths for comfort
- Improve mobile above-the-fold usability
