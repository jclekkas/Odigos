#80 - Shareable Deal Scorecard (V1)

Task:
Add a shareable scorecard to Odigos results so users can quickly share a clean summary of their analysis with a partner, friend, parent, or group chat without exposing the raw quote or sensitive details.

Objective:
After analysis completes, users should be able to:
1. copy a plain-text result summary to the clipboard
2. download a clean PNG scorecard

This is a distribution feature. It must be available in the free tier and must not require backend changes.

--------------------------------------------------
What & Why
--------------------------------------------------

Users often want a second opinion from someone they trust. Right now the only practical way to share an Odigos result is by screenshotting a dense results page. This feature creates a clean, shareable artifact that is easier to forward and easier for others to understand.

The scorecard should be simple, trustworthy, and safe:
- easy to share
- visually clean
- no sensitive quote text
- no dealer name
- no full analysis dump

--------------------------------------------------
Scope (STRICT)
--------------------------------------------------

Modify only what is needed for this feature.

Relevant files:
- client/src/pages/home.tsx
- client/src/components/SeoHead.tsx (only if needed, otherwise leave untouched)
- shared/schema.ts
- client/public/og-image.png
- client/public/favicon.png

Create:
- client/src/components/ShareCard.tsx

Allowed dependency:

Do NOT:

- log scorecard content

- redesign the whole results page

- expose specific dollar amounts in the scorecard or copied summary

--------------------------------------------------
Done Looks Like
--------------------------------------------------

After analysis results appear, two new buttons are visible in the results area:
- Copy summary
- Download scorecard

Copy summary:
- copies a clean plain-text block to the clipboard
- includes verdict, short verdict label, and up to 3 top issues
- includes Odigos footer
- includes no raw quote content
- includes no dealer name
- includes no dollar amounts
- includes no full suggested reply

Download scorecard:
- downloads a PNG named:
  odigos-deal-scorecard.png
- PNG is approximately 1200x630
- PNG contains:
  - Odigos wordmark/logo
  - color-coded verdict badge
  - verdict label
  - up to 3 top issues
  - footer: "Analyzed by Odigos • odigos.replit.app"
- PNG does NOT contain:
  - raw dealer quote
  - dealer name
  - specific dollar amounts
  - full suggested reply
  - hidden internal fields

Buttons are visible in the free tier.

The card downloads successfully in Chrome, Firefox, and Safari.

--------------------------------------------------
Implementation Tasks
--------------------------------------------------

1) Install dependency

Install:
- html-to-image

No other new dependencies.

2) Build ShareCard component

Create:
- client/src/components/ShareCard.tsx

Purpose:
Render a fixed-size card specifically for PNG export.

Requirements:
- fixed size around 1200x630
- visually clean and readable
- Odigos branding at top-left
- verdict badge with color based on goNoGo / verdict state
- verdict label shown prominently
- list up to 3 issues
- footer:
  "Analyzed by Odigos • odigos.replit.app"

Inputs:
- goNoGo
- verdictLabel
- topIssues

Optional:
- dealScore only if already available and actually displayed on the card
Do NOT require dealScore if it is not needed visually.

Issue source:
Use the best short, human-readable issue text already available from the analysis result.
Do NOT use raw/internal field keys if those are awkward for users.
If the current result shape only exposes missingInfo[].field, convert those values into readable labels before display.

Fallbacks:
- if fewer than 3 issues exist, render only available issues
- if zero issues exist, show:
  "No major issues surfaced in this summary"
- do not render empty bullets

Rendering behavior:
- component must be mounted off-screen or otherwise hidden from normal UI
- it must still be fully renderable for html-to-image capture
- do not use display:none for the captured node if that breaks image generation

3) Add share actions to results UI

File:
- client/src/pages/home.tsx

Placement:
- after the verdict/score area
- before the paywall / premium upsell section

UI:
- small horizontal row with two buttons
- labels exactly:
  - Copy summary
  - Download scorecard

Behavior:
- only show when a valid analysis result exists
- do not show before analysis completes

4) Implement Copy summary

Build a plain-text summary from result fields using this exact format:

Odigos Deal Scorecard
Verdict: [GO / NO-GO / NEED-MORE-INFO]
[verdictLabel]
Issues flagged:
• [issue 1]
• [issue 2]
• [issue 3]
Analyzed by Odigos — odigos.replit.app

Rules:
- include only available issues
- if no issues are available, replace the issue list with:
  Issues flagged:
  • No major issues surfaced in this summary
- no raw quote content
- no dealer name
- no specific dollar amounts
- no suggested reply text

Clipboard:
- use browser clipboard API
- provide lightweight user feedback after successful copy

5) Implement Download scorecard

Use html-to-image to capture the hidden ShareCard component and download it as:
- odigos-deal-scorecard.png

Requirements:
- export must be crisp on retina/high-density screens
- preserve intended card dimensions and padding
- background must render correctly
- no cut-off text
- no transparent/unstyled export bugs

Behavior:
- only allow download when result data exists
- provide lightweight user feedback if generation fails

6) Data safety guardrails

Before rendering or copying, ensure the shared artifact excludes:
- raw quote text
- dealer name
- full reply script
- exact fee amounts / dollar figures
- any user-entered text beyond safe summary fields

This is mandatory.
