#23 - Landing Page Accessibility & UX Refinement

What & Why
A focused pass on client/src/pages/landing.tsx to fix real accessibility violations and make small UX/readability improvements — without touching analyzer logic, Stripe, pricing behavior, or routing architecture.

All changes should remain tightly scoped to this file unless an existing shared component absolutely requires a small compatibility adjustment.

Done looks like
- Hero CTA buttons (“Check a Dealer Quote”, “Try a bad deal example”) are valid HTML with no nested interactive elements
- Click behavior, keyboard behavior, data-testid attributes, and existing tracking/onClick behavior are preserved
- A skip-to-content link targeting #main-content is the first focusable element rendered on the page
- The skip link is visually hidden by default and visible on focus
- <main> has id="main-content"
- Each FAQ accordion trigger has:
  - aria-expanded with correct true/false state
  - aria-controls pointing to its answer panel
- Each FAQ answer panel has a stable matching id
- Pricing cards each include a CTA button:
  - “Try Free Preview” → /analyze
  - “Get Full Review — $49” → /analyze
- Pricing cards remain visually balanced and preserve existing copy/order
- Key supporting paragraph text gets a slight contrast lift where needed
- Alternating section backgrounds are consistent and restrained across major landing sections
- No analyzer logic, Stripe flow, pricing behavior, or routing changed

Out of scope
- OpenAI prompts, scoring, rule engine
- Stripe/payment behavior
- Any new features, accounts, or workflows
- New dependencies or framework changes
- Materially rewriting copy
- Removing educational/SEO sections
- Redesigning the page layout
- Changing pricing copy or pricing structure

Tasks

1) Fix hero CTA nested interactive elements
- Replace both hero CTA patterns that currently use nested interactive elements with a single valid interactive element each
- Preserve existing styles
- Preserve existing data-testid attributes
- Preserve existing onClick / analytics / tracking behavior
- Preserve routing behavior
- Preferred approach: use shadcn Button with asChild so Button renders the anchor/link instead of nesting

2) Add skip link and main id
- Insert a skip-to-content link as the first focusable element rendered on the page, before SiteHeader
- Target: #main-content
- Use a standard visually-hidden-until-focus pattern
- Add id="main-content" to the existing <main> element

3) Fix FAQ accordion ARIA
- Add aria-expanded to each FAQ trigger, driven by the current open/closed state
- Add aria-controls on each FAQ trigger
- Add a stable id to each FAQ answer panel and ensure it matches aria-controls
- Do not change FAQ copy or visuals

4) Add CTA buttons to pricing cards
- Add one CTA button below the feature list in each pricing card
- Free card button:
  “Try Free Preview” → /analyze
- Paid card button:
  “Get Full Review — $49” → /analyze
- Preserve existing pricing copy
- Preserve existing card order
- Keep the two cards structurally balanced
- Use button variants that maintain the current visual hierarchy

5) Lift supporting text contrast
- Selectively raise a small number of key supporting paragraphs from overly dim muted text to a slightly lighter value
- Keep hierarchy intact
- Do not brighten all muted text globally
- Keep the page calm and restrained

6) Tighten section rhythm with subtle background alternation
- Apply the existing restrained alternating background treatment consistently across major landing sections
- Build on the current bg-muted/30 pattern already in use
- Do not apply this to nested cards or small subcomponents unless necessary
- Do not introduce glows, bright gradients, or flashy styling
