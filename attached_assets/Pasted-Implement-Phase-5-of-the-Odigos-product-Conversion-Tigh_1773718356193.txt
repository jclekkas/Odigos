Implement Phase 5 of the Odigos product: Conversion Tightening.

This is NOT a redesign and NOT a new feature phase.
This is a focused pass to improve clarity, trust, and conversion using the existing structure that is now in place.

## PRIMARY GOAL

Increase the likelihood that a new visitor:
- understands what Odigos does immediately
- trusts the analysis enough to try it
- understands why the paid tier is worth $49
- proceeds into the analyzer with less hesitation

## CRITICAL CONSTRAINTS

Do NOT:
- change analyzer logic
- change upload/OCR/OpenAI/Stripe behavior
- change routes or API contracts
- touch backend files
- add new features
- redesign page structure
- remove sections
- add new libraries
- change pricing amounts
- change CTA destinations
- auto-commit or auto-merge anything

## FILE SCOPE

Prefer changes limited to:
- client/src/pages/landing.tsx
- client/src/pages/home.tsx

Only touch another file if absolutely necessary for copy or micro-UX clarity, and explicitly report it.

## WHAT THIS PHASE IS

This phase is about:
- sharper wording
- clearer value framing
- stronger trust/reassurance
- better conversion psychology
- more believable sample output
- reduced hesitation

This phase is NOT about:
- visual redesign
- layout overhaul
- new components
- business logic changes

## CONTENT RULES

- Preserve Odigos tone: calm, direct, credible, non-hype
- No flashy marketing language
- No exaggerated claims
- No AI-sounding fluff
- Prefer tightening over expanding
- Prefer specifics over abstraction
- Do not rewrite simply for the sake of rewriting
- If existing wording is already strong, keep it

## HIGHEST-PRIORITY AREAS

### 1. Sample Output section (highest priority)
This is the most important conversion section on the landing page.

Goals:
- make it feel more concrete and believable
- make the red flags feel like things a real buyer would actually see
- improve perceived intelligence and usefulness of the product
- make the paid tier feel more justified by what is shown

Constraints:
- preserve existing Odigos product terminology
- preserve existing section structure
- do not turn it into a different product
- do not add fake complexity
- do not change CTA destinations

### 2. Pricing framing
Goals:
- make the difference between free and paid more obvious
- reduce hesitation around the $49 paid tier
- improve clarity of what the user gets in each tier

Constraints:
- keep prices exactly the same
- keep CTA text exactly the same unless there is an extremely strong clarity reason, in which case explain it
- keep CTA destinations exactly the same
- do not add tiers
- do not change payment flow

### 3. Trust and reassurance
Goals:
- make independence, privacy, and speed easier to notice
- reduce skepticism
- reinforce that this is a serious tool, not generic AI fluff

Areas to inspect:
- supporting lines near hero CTA
- analyzer page reassurance copy
- upload/paste guidance
- pricing/supporting copy
- nearby explanatory microcopy

### 4. Analyzer page microcopy
Goals:
- make the analyzer feel easier and safer to use
- clarify what users can paste or upload
- reduce ambiguity before they submit
- reinforce what happens after submission

Constraints:
- no logic changes
- no flow changes
- no new steps
- keep upload and paste workflows intact
