#79 - Fix Social Sharing Previews (Production-Safe)

Task:
Fix Odigos social sharing previews so that when any URL is shared (Slack, WhatsApp, iMessage, Facebook, X/Twitter), the correct Odigos branded image and metadata appear — never the old Replit placeholder.

---

Objective:
Ensure all pages output complete, consistent Open Graph and Twitter metadata using the correct Odigos branding and a valid production image.

---

Scope (STRICT):
Modify ONLY:
- client/index.html
- client/src/components/SeoHead.tsx
- client/src/lib/seo.ts
- client/src/lib/jsonld.ts (audit only if needed)
- client/public/og-image.png
- client/public/favicon.png
- create: client/public/site.webmanifest

DO NOT:
- change routing
- change business logic

- touch Stripe/payment flow
- redesign UI
- add new dependencies

---

Core Requirements:

All pages must include:
- og:title
- og:description
- og:image
- og:image:alt
- og:url
- og:type
- og:site_name
- twitter:card = summary_large_image
- twitter:title
- twitter:description
- twitter:image

All image URLs MUST:
- be absolute
- use https://odigos.replit.app
- be publicly accessible (no auth)
