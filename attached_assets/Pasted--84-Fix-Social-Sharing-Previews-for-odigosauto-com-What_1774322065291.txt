#84 - Fix Social Sharing Previews for odigosauto.com

What & Why
When odigosauto.com links are shared on iMessage, WhatsApp, Slack, email, Twitter/X, Facebook, or LinkedIn, previews currently expose old Replit URLs and outdated branding. This makes the product look unfinished and reduces trust. Fix all metadata, URL constants, and share-image references so previews consistently reflect Odigos branding and odigosauto.com.

Done looks like
- Sharing any odigosauto.com page shows odigosauto.com as the URL, never any Replit domain
- Social preview uses the correct branded image at /og-image.png
- All required OG and Twitter tags are present and correct:
  - og:title
  - og:description
  - og:image
  - og:image:alt
  - og:url
  - og:type
  - og:site_name
  - twitter:card=summary_large_image
  - twitter:title
  - twitter:description
  - twitter:image
- Canonical URLs resolve to https://odigosauto.com
- SITE_URL / OG_IMAGE constants resolve to https://odigosauto.com everywhere
- Favicon, app icons, and manifest assets are correct and do not reference any Replit domain

Out of scope
- UI redesign
- routing changes
- analyzer changes
- new dependencies

Tasks
1. Replace all Replit URL constants
- Audit and replace every instance of:
  - odigos.replit.app
  - any replit.app variant
  - any repl.co variant
- Files:
  - client/index.html
  - client/src/components/SeoHead.tsx
  - client/src/lib/seo.ts
  - any config/env fallback using Replit hostnames
- Ensure defaults use:
  - SITE_URL = https://odigosauto.com
  - og:url = https://odigosauto.com/
  - og:image = https://odigosauto.com/og-image.png

2. Ensure complete OG + Twitter coverage
- Audit index.html and SeoHead.tsx
- Add any missing required tags
- Ensure og:site_name = "Odigos"
- Ensure twitter:card = summary_large_image

3. Replace or confirm social image
- If client/public/og-image.png is already on-brand, keep it
- Otherwise regenerate/replace it using attached_assets/logo_1771101409527.png
- Save final asset as client/public/og-image.png

4. Verify favicon + manifest assets
- Confirm:
  - client/public/favicon.png
  - client/public/icon-192.png
  - client/public/icon-512.png
  - client/public/site.webmanifest
- Ensure assets are branded correctly and do not reference any Replit domain

5. Verify canonical URLs
- Confirm canonical tags point to https://odigosauto.com paths, not Replit URLs
