Goal:
Improve the visual credibility and conversion rate of Odigos by upgrading the UI/UX styling only — without changing pricing, routing, Stripe logic, or core content meaning.

This is a UI polish pass, not a product rewrite.

⸻

🔒 Scope Rules (Do Not Break These)
	•	❌ Do NOT modify Stripe logic
	•	❌ Do NOT modify pricing or checkout code
	•	❌ Do NOT modify backend or API logic
	•	❌ Do NOT add fake metrics or unverifiable stats
	•	❌ Do NOT add new product features (no OCR, no scanning, no new workflows)
	•	❌ Do NOT change routing
	•	❌ Do NOT rename product concepts (no “terminal,” no “execute,” no “v2.4” language)

Allowed files:
	•	client/src/pages/landing.tsx
	•	client/src/pages/out-the-door-price.tsx (optional for consistency)
	•	global styling file (only if needed for background/typography updates)

⸻

🎯 Objective

Make the site feel:
	•	Trustworthy
	•	Clean
	•	Modern SaaS
	•	Premium but simple
	•	Consumer-friendly (not hacker-y or sci-fi)

We want:
“Professional finance tool”
Not:
“AI black-box experiment”

⸻

🎨 Design Direction

1️⃣ Background & Visual Tone
	•	Add subtle dark gradient background (deep navy → charcoal)
	•	Soft vignette or radial glow behind hero
	•	No animated effects
	•	No neon hacker aesthetics

2️⃣ Typography Hierarchy
	•	Increase H1 size and spacing
	•	Improve subheading readability
	•	Ensure strong contrast on text
	•	Reduce cramped vertical spacing

Hero structure should clearly read:
	•	Logo
	•	Headline
	•	Subheadline
	•	Primary CTA
	•	Compact trust strip

⸻

3️⃣ CTA Upgrade

Primary CTA should:
	•	Be larger
	•	High contrast
	•	Clear wording:
	•	“Analyze My Quote”
OR
	•	“Check My Dealer Messages”

Button styling:
	•	Rounded
	•	Soft gradient or strong solid color
	•	Clear hover state
	•	No glow animations

⸻

4️⃣ Trust Strip (Critical for Conversion)

Directly under the hero CTA, add a compact trust row:

Example format:
Independent • No account required • We don’t store your text • Secure Stripe checkout

Muted text, small size, but visible without scrolling.

This is more important than visual flair.

⸻

5️⃣ Card & Section Polish

All content sections should:
	•	Have consistent padding (increase breathing room)
	•	Use subtle card containers where appropriate
	•	Soft borders or light shadows
	•	Clear section dividers

Do NOT remove content sections.
Do NOT change copy meaning.

⸻

6️⃣ Remove Any “Sketchy” Signals

Avoid:
	•	“Execute analysis”
	•	“Terminal”
	•	“Predatory detection ON”
	•	Fake statistics
	•	Big unverifiable claims

This must feel grounded and real.

⸻

✅ Acceptance Criteria

Before finishing, confirm:
	•	Stripe checkout still works
	•	All routes work
	•	No console errors
	•	Mobile layout looks clean
	•	CTA is visible above the fold
	•	Trust strip visible above the fold
	•	No fake metrics added
	•	No new features added

After implementing, list:
	•	Files modified
	•	Exact changes made
	•	Confirmation that Stripe logic was untouched
