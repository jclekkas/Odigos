Build a very simple web app prototype called “Odigos” that helps a user evaluate a car purchase offer BEFORE visiting a dealership.

Scope is intentionally small. Prioritize a working end-to-end flow over perfect parsing or design.

Tech:
- Simple Next.js app (TypeScript preferred)
- One page is fine
- Use an OpenAI-compatible LLM call behind a single server endpoint or server action
- Minimal dependencies

UI (single page):
1) Header: “Odigos — Know your car deal before you go in”
2) Required input:
   - Large textarea labeled: “Paste dealer texts, emails, or quotes here”
3) Optional inputs (collapsed section titled “Optional details (improves accuracy)”):
   - New or Used (dropdown: Unknown/New/Used)
   - Vehicle (free text: year/make/model/trim)
   - ZIP code (text)
   - Purchase type (dropdown: Unknown/Cash/Finance/Lease)
   - If Finance: APR (number), term months (number), down payment (number)
4) Button: “Analyze Deal”
5) Output area below results with clear section headings and copy-to-clipboard for the suggested reply

Behavior:
When the user clicks “Analyze Deal”:
- Send the pasted dealer text + any optional fields to the LLM.
- The LLM must return a structured JSON object (so we can render cleanly), plus a readable summary.

Critical requirement:
If key context is missing or ambiguous (e.g., no out-the-door price, unclear fees, unclear whether new/used, only monthly payment mentioned), Odigos must explicitly say what is missing and ask for the exact next question(s) the user should send to the dealer.

Output (must be decisive and actionable):

A) DEAL SCORE (very prominent)
- One of: GREEN / YELLOW / RED
- Also include: “GO / NO-GO” for whether the user should go to the dealership now

B) WHAT THIS DEAL LIKELY MEANS (plain English)
- Summarize the offer and what’s included/excluded
