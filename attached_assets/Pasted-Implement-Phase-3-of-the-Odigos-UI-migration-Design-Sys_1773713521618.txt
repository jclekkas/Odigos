Implement Phase 3 of the Odigos UI migration: Design System Alignment.

This phase is a controlled visual-system adjustment to close the gap between the current Replit implementation and the v0/Vercel prototype.

## PRIMARY GOAL

Bring the app’s overall visual tone closer to the v0 prototype by introducing a warmer, cream-toned background/surface system and aligning the analyzer page with the upgraded landing-page aesthetic.

This phase is about visual system alignment only.
Do NOT change product behavior, routes, business logic, pricing behavior, or page structure.

## CONTEXT

Phases 1, 2A, and 2B are complete.
The landing page structure has been migrated, but there is still a noticeable gap versus the v0 prototype in:
- background tone
- surface warmth
- global visual cohesion
- analyzer-page visual consistency

The main mismatch now is that the current app still uses a colder neutral design system, while the v0 prototype uses a warmer cream/off-white background and softer surfaces.

## ALLOWED SCOPE

You MAY modify only what is necessary to align the visual system safely, including:
- global color tokens / theme variables
- carefully selected global CSS values
- analyzer page presentation styles/classes
- landing/analyzer surface/background classes where needed

Prefer the smallest possible set of changes that produces a clear shift toward the v0 feel.

## DESIGN TARGET

Match the v0 prototype’s overall feel more closely by introducing:
- a warm cream/off-white page background instead of the current colder neutral background
- softer section/surface contrast
- consistent premium/editorial tone across landing + analyzer
- preserved readability and accessibility

The goal is not to hardcode random hex colors throughout the app.
The goal is to update the design system cleanly and minimally.

## IMPLEMENTATION RULES

- Prefer token-based/global-theme changes over scattered one-off overrides
- Do not introduce inconsistent ad hoc colors
- Preserve accessibility and contrast
- Preserve dark-mode safety if relevant, or explicitly avoid breaking it
- Keep the diff reviewable and intentional
- Align the analyzer page visually with the landing page, but do not alter its structure or behavior
