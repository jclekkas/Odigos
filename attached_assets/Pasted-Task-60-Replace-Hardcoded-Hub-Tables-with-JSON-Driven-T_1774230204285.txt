Task #60: Replace Hardcoded Hub Tables with JSON-Driven Tables
Context
Both hub pages (/car-dealer-fees-by-state and /dealer-doc-fee-by-state) have hardcoded tables with only 20 and 30 states respectively. The authoritative data source is server/state_fee_reference.json, which contains verified data for all 50 states + DC. The tables need to be generated from this JSON so they're always in sync with the data the analyzer uses.
There are also factual errors in the current hardcoded tables — most notably Texas is listed as "No cap" when it's actually capped at $225. Rebuilding from the JSON fixes these automatically.
What to do
Step 1: Create a shared client-side data module
Create client/src/data/stateReferenceData.ts that:

Imports or embeds the data from server/state_fee_reference.json (the same dataset already used by the analyzer)
Exports a typed array of all 51 entries (50 states + DC), sorted alphabetically by state name
Each entry should expose: name, abbreviation, docFeeCap (boolean), docFeeCapAmount (number or null), docFeeTypicalRange ([low, high]), stateSalesTaxRate, tradeInTaxCredit
Export a helper function formatDocFeeDisplay(state) that returns a human-readable string:

If capped: "Up to $[capAmount] (capped)"
If uncapped: "[low]–[low]–
[low]–[high] (no cap)"




Step 2: Rebuild the table in client/src/pages/car-dealer-fees-by-state.tsx

Replace the hardcoded <table> rows with a .map() over all 51 entries from the data module
Columns: State Name | Typical Doc Fee Range (use formatDocFeeDisplay)
Sort alphabetically by state name
Each state name should link to its state page if one exists (/car-dealer-fees-[slug]). For states without pages yet, render plain text (not a link)
Keep all surrounding page copy, structure, and SEO content exactly as-is — only replace the table contents
The 5 existing state page links (California, Texas, Florida, New York, Georgia) in the "State-specific dealer fee guides" section at the bottom should remain unchanged

Step 3: Rebuild the table in client/src/pages/dealer-doc-fee-by-state.tsx

Same approach: replace hardcoded rows with a .map() over all 51 entries
Columns: State Name | Doc Fee Cap? | Typical Doc Fee
"Doc Fee Cap?" column: "Capped at $[amount]" if capped, "No cap" if uncapped
"Typical Doc Fee" column: the range from docFeeTypicalRange
Sort alphabetically
Keep all surrounding page copy unchanged

Step 4: Verify consistency
After both tables are rebuilt, confirm:

Both tables show exactly 51 rows (50 states + DC)
Texas shows as "capped at $225" (NOT "no cap")
Pennsylvania shows as "capped at ~$490" (2026 figure)
Illinois shows as "capped at ~$378"
Maryland shows as "capped at $800"
Michigan shows as "capped at $280"
Ohio shows as "capped at $387"
Minnesota shows as "capped at $350"
Both tables produce identical data for the same state

Do NOT

Modify the JSON data source itself
Change surrounding page copy, SEO meta, or page structure
Add new columns beyond what's described
Create a separate copy of the state data — always read from one source
Modify client/src/data/stateFees.ts or the dynamic state page template as part of this task
