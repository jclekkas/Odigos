#83 - "Deals Analyzed" Public Counter

What & Why
There is no visible signal to users that real people are using this tool. A simple public counter — "X real deals analyzed" — adds credibility and social proof at the moment a user is deciding whether to trust the product.

Done looks like
A public /api/stats/count endpoint returns { count: number } without requiring any admin key.
The count is sourced from core.platform_metrics.real_deals_analyzed, not the legacy metrics event log.
This metric aligns with warehouse truth logic: only rows where counts_toward_real_deals = true, excluding seed data, CFPB/public complaint rows, duplicates, test data, and pipeline-error rows.
If core.platform_metrics is empty or unavailable, the endpoint falls back to a direct aggregate:
SELECT COUNT(*) FROM core.listings WHERE counts_toward_real_deals = true.
The landing page shows a small "X real deals analyzed" pill or line of text directly below the hero section, fetched live via this endpoint.
The analyzer page shows the same counter near the top of the input form, using the same query key (shared/cached).
Both counters show a skeleton/placeholder while loading and fail silently (hide entirely) if the request errors.
The number is displayed formatted with commas (e.g. "1,247 real deals analyzed").

Out of scope
Any other stats
Admin-only views or dashboards
Real-time polling or websocket updates

Tasks
Public stats endpoint
- Add GET /api/stats/count to server/routes.ts
- Query core.platform_metrics and return { count: real_deals_analyzed }
- If the view is empty or unavailable, fall back to:
  SELECT COUNT(*) FROM core.listings WHERE counts_toward_real_deals = true
- No auth required

Landing page counter
- In client/src/pages/landing.tsx, fetch /api/stats/count with TanStack Query
- Render a subtle "X real deals analyzed" badge below the hero headline
- Hide on error; show skeleton while loading

Analyzer page counter
- In client/src/pages/home.tsx, use the same /api/stats/count query key near the top of the input form so the result is shared and cached
